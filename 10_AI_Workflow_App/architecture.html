ScriptSprout

Full-stack request flow. The browser talks to a FastAPI ASGI app (create_app() in ScriptSprout/backend/app/main.py), usually run with Uvicorn in dev from ScriptSprout/backend/. Downstream: services, OpenAI SDK, SQLite, ChromaDB. Paths below are under ScriptSprout/ (runnable app root).

Thin path

ScriptSprout/backend/app/db/repos/* from routes — lists, detail, some admin reads without a heavy service wrapper.
Thin path API → app/db/repos → SQL (above right): repo-centric reads/writes without a heavy service wrapper. Services path: generative workflows, indexing, and anything that coordinates OpenAI + persistence (services still use SQLAlchemy/repos internally where needed). One route handler may call both repos and services in the same request. No Agents SDK and no OpenAI function-tool loop — control flow is product-defined Python.

What this emphasizes

  • HITL workflow and guardrails in a real web app (not a CLI).
  • Clear split: routes vs services vs repos vs stores.
  • Same teaching spirit as chapter 9 — orchestrated workflow, not an autonomous multi-tool agent.
